## 우리가 해결하는 문제

디지털 광고에서 크리에이티브 팀은 매일 도전에 직면합니다: 수십 개의 서로 다른 플랫폼이 요구하는 정확한 사양에 모든 에셋이 부합하는지 확인하는 것입니다. 크기가 잘못된 크리에이티브 하나가 광고 거부, 캠페인 예산 낭비, 시장 기회 손실로 이어질 수 있습니다.

기존 워크플로우의 문제점:
- 흩어진 문서를 참고하며 수동으로 크기 확인
- 각 플랫폼에 맞는 끝없는 리사이즈 변형 제작
- 어떤 클라이언트가 어떤 포맷을 필요로 하는지 추적 불가
- 정리되지 않은 폴더 구조에서 검색
- 브랜드 가이드라인과 규정 준수 요구사항 누락

이러한 비효율성은 에이전시와 마케팅 팀에게 매년 수천 시간의 비용을 초래합니다.

## 우리의 솔루션

Creative Asset Validator는 크리에이티브 에셋 워크플로우 전체를 자동화하는 종합 플랫폼입니다. 마케팅 팀, 에이전시, 기업을 위해 설계되었으며, 플랫폼 사양에 따라 에셋을 검증하고, 지능적으로 정리하며, AI를 활용해 크리에이티브 성과를 최적화합니다.

선택적 클라우드 동기화와 함께 브라우저에서 완전히 실행되며, 복잡한 백엔드 인프라 없이도 엔터프라이즈급 보안과 협업 기능을 제공합니다.

## 핵심 기능

### 에셋 라이브러리 및 검증

모든 크리에이티브 에셋의 중앙 허브. 드래그 앤 드롭으로 이미지와 동영상을 업로드하면, 시스템이 YouTube, TikTok, Meta, Google Ads, DV360, The Trade Desk, 커넥티드 TV 네트워크 등 50개 이상의 광고 플랫폼 사양에 맞춰 자동으로 검증합니다.

- 자동 크기 및 포맷 검증
- 파일 크기 및 길이 규정 준수 확인
- 각 플랫폼별 시각적 상태 표시
- 태그와 즐겨찾기로 폴더 정리
- 개인 및 팀 저장소 분리

### AI 기반 크리에이티브 분석

여러 AI 제공업체를 활용하여 크리에이티브 효과를 분석합니다:

- **훅 분석**: 첫 몇 초의 주의 집중력 평가
- **CTA 감지**: 행동 유도 문구의 명확성과 배치 평가
- **브랜드 규정 준수**: 로고 배치, 색상, 가이드라인 확인
- **성과 예측**: 참여율, CTR, 전환 잠재력 추정

Claude, GPT-4, Gemini를 조율하여 다양한 관점에서 포괄적인 분석을 제공합니다.

### 브랜드 키트 생성기

하나의 로고를 완전한 브랜드 에셋 패키지로 변환합니다. 고해상도 로고 하나를 업로드하면 100개 이상의 포맷 변형을 생성합니다:

- 소셜 미디어 프로필 이미지 및 커버
- Google Ads 배너 크기
- 파비콘 및 앱 아이콘
- 이메일 서명
- 인쇄용 포맷

각 변형은 적절한 종횡비를 유지하며 저해상도 소스에 대한 AI 업스케일링을 포함합니다.

### AI Studio

AI 생성 기능에 대한 직접 접근:

- **텍스트-이미지**: Gemini를 사용하여 설명에서 비주얼 생성
- **이미지-비디오**: Veo 3.1로 정지 이미지를 동영상으로 변환
- **아웃페인팅**: 이미지를 새로운 종횡비로 확장
- **배경 제거**: 즉시 피사체 분리

### 전략 모듈

자신감 있게 캠페인을 계획하세요:

- **배치 매트릭스**: 모든 플랫폼 요구사항을 보여주는 시각적 그리드
- **파생 로드맵**: 마스터 에셋에서 필요한 모든 변형 매핑
- **A/B 테스트 플래너**: 가설 기반 크리에이티브 테스트 설계
- **크리에이티브 피로도 예측**: 에셋 갱신 시기 파악

### CRM 통합

별도 도구가 필요 없는 내장 고객 및 프로젝트 관리:

- 브랜드 가이드라인이 포함된 회사 프로필
- 타임라인과 예산이 포함된 프로젝트 추적
- 에셋 연결 및 버전 이력
- 세분화된 접근 수준의 팀 공유 (보기, 편집, 전체)
- 도메인 기반 자동 공유

### 통합 허브

기존 에코시스템에 연결:

- **Google Drive**: 폴더 탐색, 에셋 스캔, 직접 가져오기
- **Google Sheets**: 스프레드시트에서 이미지 URL 추출
- **Gmail**: 첨부 파일에서 크리에이티브 에셋 스캔
- **Dropbox**: 폴더 동기화
- **Slack**: 채널 파일 스캔

각 통합에는 어떤 에셋이 요구사항을 충족하고 어떤 에셋이 주의가 필요한지 보여주는 적격성 분석이 포함됩니다.

## 아키텍처

### SaaS-Ready 인프라

프로덕션 배포를 위한 완전한 백엔드 아키텍처를 포함합니다:

**데이터베이스 레이어 (MySQL)**
- 역할 기반 접근 제어가 있는 사용자 및 팀 관리
- 검증 결과가 포함된 에셋 메타데이터
- CRM 엔티티 (회사, 프로젝트, 연락처)
- 오프라인 우선 운영을 위한 동기화 추적
- 사용 분석 및 감사 로깅

**API 레이어 (PHP)**
- 모든 작업을 위한 RESTful 엔드포인트
- Google OAuth 토큰 검증
- 암호화된 세션 관리
- 에셋 처리를 위한 Cloudinary 통합

**동기화 엔진 (JavaScript)**
- 브라우저와 서버 간 양방향 동기화
- 버전 추적을 통한 충돌 해결
- 보류 중인 변경사항을 위한 오프라인 큐
- 실시간 상태 업데이트

**저장소 옵션**
- 로컬: 브라우저 기반 저장을 위한 IndexedDB
- 클라우드: 이미지/비디오 처리 및 CDN 배포를 위한 Cloudinary
- 하이브리드: 로컬과 클라우드 간 자동 동기화

### 보안 구현

| 레이어 | 구현 |
|-------|----------------|
| 세션 암호화 | PBKDF2 키 파생을 사용한 AES-256-GCM |
| 인증 | JWT 검증이 포함된 Google SSO |
| 도메인 적용 | 구성 가능한 기업 도메인 요구사항 |
| 기기 바인딩 | 브라우저 핑거프린트에 연결된 세션 |
| 위변조 방지 | HMAC-SHA256 서명 검증 |
| 활동 로깅 | 내보내기 가능한 360일 감사 추적 |
| 데이터 격리 | 사용자별 암호화 저장소 |
| 역할 기반 접근 | 슈퍼 관리자, 도메인 관리자, 편집자, 뷰어 |
